Picking Priority Agent for Odoo Warehouse Management

Smart Warehouse Prioritization with AI Copilot

Compatible with Odoo 16, 17, 18, 19 Community and Enterprise OpenRouter, OpenAI, Google Gemini

Picking Priority Agent automatically prioritizes warehouse pickings using intelligent rule-based scoring with optional AI Copilot assistance. The system evaluates SLA deadlines, stock availability, order value, dependencies, and warehouse complexity so teams can focus on the most critical work first.

Overview

Instead of manually deciding which orders to pick first, the system calculates a priority score and recommended action for each picking. That keeps warehouse operations fast, consistent, and easy to manage.

Result: Faster operations, improved SLA compliance, and reduced manual decision-making.

How Priority Scoring Works

The system calculates a priority score using multiple operational factors.

Each factor contributes a configurable number of points toward the total priority score.

By default, the system installs with the following recommended configuration:

Factor Default Max Points Description
Deadline Pressure30Orders nearing or exceeding SLA deadlines receive higher priority.
Stock Availability20Fully reserved stock receives the highest score. Partial or unavailable stock lowers the score.
Manual Urgency15Managers can flag orders as High or Critical urgency.
Source Channel10Marketplace orders usually receive higher priority than B2B or internal transfers.
Downstream Dependencies10Orders blocking other operations receive higher priority.
Business Value10Higher-value orders receive higher priority.
Picking Complexity5Simple picks with fewer products and zones receive a small bonus.
Total possible score: 100 points

Fully Configurable Scoring System

These values are default recommendations only.

Warehouse managers can fully customize:

  • Factor weights
  • Maximum score per factor
  • Enabled or disabled factors
  • Company-specific configurations
  • Warehouse-specific configurations

All settings can be adjusted in:

Inventory - Configuration - Picking Priority - Configuration

This allows each business to adapt the prioritization logic to match their own operational rules.

Examples of customizations:

  • Increase Stock Availability weight for fast-moving warehouses
  • Increase Order Value importance for B2B operations
  • Reduce Complexity bonus for large warehouses
  • Prioritize Marketplace channels over internal transfers

Changes apply immediately to all priority calculations.

Installation

  1. Copy the module folder into your Odoo custom addons directory.
  2. Activate Developer Mode in Odoo.
  3. Go to Apps - Update Apps List.
  4. Search for Picking Priority Agent.
  5. Click Install.
  6. Go to Inventory - Configuration - Picking Priority.
  7. Optionally configure AI from Inventory - Configuration - AI Configuration.

Daily Usage

  • Warehouse Staff: Open Inventory - Operations - Transfers and work on Critical and High items first.
  • Supervisors: Override priorities, review audit logs, and analyze priority breakdowns.
  • Managers: Configure priority policies, use AI Policy Builder, and analyze warehouse performance.

Frequently Asked Questions

Can this module work without AI? Yes. The rule-based prioritization works completely without AI. AI features are optional enhancements.

Does it support Odoo Community? Yes. Both Community and Enterprise editions are supported.

Can I switch AI providers later? Yes. You can change providers anytime from the configuration menu.

Will AI change my data? No. AI only provides recommendations and explanations. All priority scoring remains deterministic and configurable.
